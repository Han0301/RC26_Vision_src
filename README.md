# RC26 Vision src — 上车融合代码工作区

> 多传感器融合感知框架，从 merge_ws20 到 rc_ws2 共 12 个版本的完整迭代。

## 项目迭代背景

`merge_ws` 系列是 RC26 竞赛机器人的 **上车运行融合代码**。早期在 camera_ws 和 world_ws 中分别验证了相机检测和 Z-buffer 感知算法，merge_ws 负责将这些算法整合到统一的 ROS 节点中，在真实 RoboMaster / RoboCup 机器人上运行，并从最初的单传感器简单融合逐步演进为多传感器、多通道、具备上位机交互的完整竞赛系统。

## 编译

```bash
catkin_make -j2
source devel/setup.bash
roslaunch merge merge.launch
```

## 版本迭代日志

### merge_ws20 — 上车初始版
**问题**: 仿真算法需要整合到上车框架，传感器数据到控制指令的链路尚未打通。
**方案**: 搭建 ROS 融合节点框架，接入 livox 激光雷达驱动，实现基础点云处理与线程池管理。

### merge_ws21 — 标定模块与线程池重构
**问题**: 上车后发现传感器外参未标定导致数据对齐偏差；线程池在真实场景下存在任务堆积风险。
**方案**: 新增相机-激光雷达外参标定模块；线程池增加背压机制，防止内存暴涨。

### merge_ws22 — 遮挡处理与 Debug 构建
**问题**: 真实环境中遮挡情况比仿真复杂，模型误检率高；上车调试困难。
**方案**: 新增 Z-buffer 遮挡处理模块过滤假阳性检测；切换 Debug 构建模式便于调试。

### merge_ws33 — 视觉里程计与点云配准
**问题**: 轮式里程计在复杂地形下打滑，定位漂移严重；点云配准精度不足。
**方案**: 新增视觉里程计与轮式里程计融合；引入 nano_gicp 轻量级 GICP 配准。

### merge_ws39 — LiDAR 目标识别与上层决策
**问题**: 激光雷达仅用于定位建图，未用于目标识别；多传感器输出缺乏统一决策层。
**方案**: 新增 LiDAR 目标识别模块；新增上层决策层统一管理视觉/激光/Z-buffer 三通道输出。

### merge_ws39.5 — ORB 特征匹配与后处理增强
**问题**: 视觉定位在纹理贫乏或光照变化场景下容易丢失；决策融合后存在异常跳动。
**方案**: 引入 ORB 特征匹配增强视觉定位鲁棒性；新增上层后处理层平滑输出。

### merge_ws39.6 — 感知与控制管线优化
**问题**: 长时运行时相机丢帧、控制抖动、坐标漂移。
**方案**: 优化相机缓存机制减少丢帧；优化控制参数提升运动平稳性；重构坐标变换减少累计误差。

### merge_ws39.7 — PnP 位姿估计与参数配置系统
**问题**: 抓取目标需要精确 6D 位姿；大量硬编码参数上车调试效率低。
**方案**: 新增 PnP 位姿估计模块；新增参数配置系统支持运行时动态调整；新增 R1/R2 差异化控制。

### merge_ws39.75 — 感知管线细化与稳定化
**问题**: 系统在长时间运行中出现性能衰减和累积漂移。
**方案**: 细化 camera/control/lidar 各模块参数；加固坐标变换异常检测；微调融合策略。

### merge_ws39.8 — 上位机集成与最终优化
**问题**: 竞赛需要上位机远程监控与指令下发；全链路尚未完全调通。
**方案**: 新增上位机交互逻辑；完善参数持久化；确定融合最终权重。

### rc_ws1 — 竞赛候选版
**问题**: 系统需要通过竞赛级别的可靠性验证。
**方案**: 全链路系统集成测试，修复边缘 case。

### rc_ws2 — 竞赛最终版
**问题**: 竞赛需要设备自检、双雷达热备、挑战赛路径管理等运行期保障。
**方案**: 新增设备自检系统、双雷达重启管理、挑战赛路径配置；新增竞赛文档体系。
