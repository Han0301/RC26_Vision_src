# RC26 Vision src — 上车融合代码工作区

> 多传感器融合感知框架，整合相机、激光雷达、Z-buffer 遮挡处理、YOLO 检测等模块，运行在 RC26 竞赛机器人上。

## 项目背景

`merge_ws` 系列是 **上车运行的融合代码**，负责将仿真中验证的算法（camera_ws 的相机检测、world_ws 的 Z-buffer 感知、yolo 模型推理）整合到统一的 ROS 节点中，在真实机器人上运行。版本后缀的数字代表迭代序号。

## 快速开始

```bash
cd merge_ws
catkin_make -j2
source devel/setup.bash
roslaunch merge merge.launch
```

## 版本迭代日志

### merge_ws20 — 上车初始版
**背景**: 首次将仿真算法整合到上车代码框架中，需要打通从传感器数据到控制指令的完整链路。

**功能改进**:
- 搭建 ROS 融合节点框架，统一管理相机、激光雷达、里程计数据
- 整合 livox_ros_driver2 激光雷达驱动
- 实现基础的点云处理管线（体素滤波、平面提取）
- 引入线程池 (`threadpool.h`) 管理并行任务
- 新增 shell_monitor 脚本监控系统运行状态

### merge_ws21 — 标定模块与线程池重构
**背景**: 上车后发现传感器外参未标定导致数据对齐偏差；线程池在真实场景下存在任务堆积风险。

**功能改进**:
- **新增标定模块** (`calibration.cpp/h`): 相机到激光雷达的外参标定，解决多传感器数据对齐问题
- **线程池重构**: 新增任务队列背压机制 (`_TREADPOOL_FLAG_`)，防止任务堆积导致内存暴涨
- 优化坐标变换管线 (`coordinate.h`)，统一多传感器坐标系
- 改进点云滤波参数，适应真实环境噪声

### merge_ws22 — 遮挡处理与 Debug 构建
**背景**: 真实环境中遮挡情况比仿真更复杂，需要专门的遮挡处理模块；同时上车调试困难，需要更好的调试支持。

**功能改进**:
- **新增遮挡处理模块** (`occlusion_handing.h`): 基于 Z-buffer 原理判断目标是否被遮挡，过滤假阳性检测
- **切换 Debug 构建模式**: `CMAKE_BUILD_TYPE=Debug`，方便上车调试时输出完整的日志和栈信息
- 优化合并决策逻辑，融合多传感器检测结果时考虑遮挡置信度

### merge_ws33 — 视觉里程计与点云配准
**背景**: 仅靠轮式里程计在复杂地形下容易打滑，需要视觉辅助定位；同时需要更精确的点云配准来建立地图。

**功能改进**:
- **新增视觉里程计** (`visual_odometry.h`): 利用相机图像估计帧间运动，与轮式里程计融合
- **引入 nano_gicp**: 轻量级 GICP 点云配准库，替代传统 ICP，提升配准精度和速度
- 新增 `lidar.h/lidar.cpp` 中激光雷达数据处理独立模块
- 重构点云地图构建管线，支持增量式地图更新
- 优化全局变量管理，减少跨文件耦合

### merge_ws39 — LiDAR 感知与上层决策
**背景**: 之前的版本只用了激光雷达做定位和建图，没有利用其做目标识别；同时需要上层决策层来协调各模块的输出。

**功能改进**:
- **新增 LiDAR 目标识别** (`lidar/lidar_recognition.h`, `lidar/lidar_getttf.h`): 基于激光雷达点云的目标检测与分类，与视觉检测形成冗余互补
- **新增上层决策层** (`superstratum/super.h`, `superstratum/super2.h`): 统一管理视觉、激光、Z-buffer 三个感知通道的输出，做最终决策融合
- **新增数据滤波模块** (`filter.h`, `fliter.cpp`): 多传感器数据的时间同步与空间滤波，减少噪声干扰
- **新增 YOLO 模型变体** (`yolo/yolo_han2.h`): 适配更多检测场景
- 重构 `camera.cpp` 和 `lidar.cpp` 的接口，统一感知模块调用方式
