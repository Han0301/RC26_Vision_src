# 无硬件环境下的串口测试方法

> 没有下位机（单片机/嵌入式设备）时，Linux 提供多种虚拟串口方案用于测试 `Ten_serial` 模块。

---

## 方案一：`socat` 虚拟串口对（最推荐）

`socat` 创建一对互联的虚拟串口，一端给程序用，另一端模拟下位机。

### 安装

```bash
sudo apt install socat
```

### 创建虚拟串口对

```bash
socat -d -d pty,raw,echo=0 pty,raw,echo=0
```

输出示例：
```
2026/07/03 10:00:00 socat[12345] N PTY is /dev/pts/2
2026/07/03 10:00:00 socat[12345] N PTY is /dev/pts/3
```

`/dev/pts/2` 和 `/dev/pts/3` 已连通——一端写入，另一端可读到。

### 测试步骤

**终端 1** — 运行你的 ROS 节点（使用 `/dev/pts/2`）：

```cpp
// 修改 GetInstance 参数
auto& serial = Ten::Ten_serial::GetInstance("/dev/pts/2", 115200);
```

**终端 2** — 监听另一端，看程序发了什么：

```bash
cat /dev/pts/3 | xxd
```

**终端 3** — 模拟下位机发送数据：

```bash
# 构造一帧：AA 55 frame_id length data... XOR EE
# 假设 frame_id=1, length=4 (4字节float), 数据全零
printf '\xAA\x55\x01\x04\x00\x00\x00\x00\x04\xEE' > /dev/pts/3
```

### 一键测试脚本

```bash
#!/bin/bash
# test_serial_pair.sh
socat -d -d pty,raw,echo=0 pty,raw,echo=0 2>&1 | tee /tmp/socat.log &
SOCAT_PID=$!
sleep 1

PTY1=$(grep -oP 'PTY is \K(/dev/pts/\d+)' /tmp/socat.log | sed -n '1p')
PTY2=$(grep -oP 'PTY is \K(/dev/pts/\d+)' /tmp/socat.log | sed -n '2p')

echo "程序使用: $PTY1"
echo "测试终端: $PTY2"
echo ""
echo "在另一个终端执行: cat $PTY2 | xxd   # 看接收"
echo "或: echo ... > $PTY2               # 模拟发送"

wait $SOCAT_PID
```

## 方案二：Python 虚拟串口测试工具

```python
#!/usr/bin/env python3
"""serial_test_harness.py — 完整的串口模拟测试工具"""
import subprocess
import re
import serial
import struct
import time

# 1. 创建虚拟串口对
proc = subprocess.Popen(
    ['socat', '-d', '-d', 'pty,raw,echo=0', 'pty,raw,echo=0'],
    stderr=subprocess.PIPE
)

lines = [proc.stderr.readline().decode() for _ in range(4)]
pts = re.findall(r'/dev/pts/\d+', '\n'.join(lines))
DEV_APP, DEV_TEST = pts[0], pts[1]
print(f"程序端: {DEV_APP}")
print(f"测试端: {DEV_TEST}")

# 2. 打开测试端串口
ser_test = serial.Serial(DEV_TEST, 115200, timeout=0.1)

# 3. 模拟下位机：循环接收并打印
print("\n等待数据... (按 Ctrl+C 退出)")
try:
    while True:
        if ser_test.in_waiting:
            data = ser_test.read(ser_test.in_waiting)
            print(f"收到 {len(data)} 字节: {data.hex()}")
            
            # 如果是有效帧，回复确认
            if len(data) >= 2 and data[0] == 0xAA and data[1] == 0x55:
                print("  → 检测到有效帧头!")
                # 构造回复帧
                response = bytes([0xAA, 0x55, 0xFF, 0x01, 0x00, 0x00, 0xEE])
                ser_test.write(response)
                print(f"  回复: {response.hex()}")
        time.sleep(0.01)
except KeyboardInterrupt:
    print("\n退出")
finally:
    ser_test.close()
    proc.terminate()
```

## 方案三：`/dev/ttynull` 快速验证

Linux 提供 `/dev/ttynull`——一个永远成功的虚拟串口（读写无实际效果），适合验证**初始化不崩溃**和**发送不抛异常**。

```cpp
// 快速验证代码
auto& serial = Ten::Ten_serial::GetInstance("/dev/ttynull", 115200);
if (serial.isOpen()) {
    std::cout << "✅ 串口打开成功" << std::endl;
    float data[4] = {1.0, 2.0, 3.0, 4.0};
    size_t sent = serial.serial_send(data, 1, sizeof(data));
    std::cout << "发送 " << sent << " 字节" << std::endl;  // 输出 22
}
```

> 注意：`/dev/ttynull` 写入的数据被丢弃，读取永远返回空。仅用于验证初始化逻辑和发送不崩溃。

## 方案四：Python `os.openpty()` 纯 Python 方案

```python
#!/usr/bin/env python3
"""使用 Linux PTY 创建虚拟串口对（无需 socat）"""
import os
import tty

master, slave = os.openpty()
tty.setraw(master)

name_master = os.ttyname(master)
name_slave  = os.ttyname(slave)

print(f"程序端: {name_slave}")     # 给 Ten_serial 用
print(f"测试端: {name_master}")    # 用 Python 读写

# 在测试端发送数据
os.write(master, b'\xAA\x55\x01\x04\x00\x00\x00\x00\x04\xEE')

# 读取程序端发来的数据
data = os.read(master, 1024)
print(f"收到: {data.hex()}")
```

## 方案五：硬件回环测试

如果有 USB 转串口模块（CH340 / CP2102 / FT232）：

```bash
# 1. 查看设备
ls -l /dev/ttyUSB*

# 2. 短接 TX 和 RX 引脚（用杜邦线）
#    发送的数据会被自己收到

# 3. 运行程序
#    用一个线程 serial_send，另一个线程 serial_read2
#    验证能否收到自己发的数据
```

## 推荐测试流程

```
第1步: /dev/ttynull
  → 验证初始化不崩溃 + 发送函数不抛异常

第2步: socat 虚拟串口对
  → 验证实际收发 + 帧协议正确性
  → 用 xxd 或 Python 脚本观察原始字节

第3步: Python 自动化脚本
  → 发送已知帧 → 验证 serial_read2 能否正确解析
  → 发送损坏帧 → 验证状态机能正确恢复

第4步: 硬件回环
  → 验证真实串口驱动

第5步: 接入真实下位机
  → 系统联调
```

大部分问题在第 2 步就能发现（帧结构错误、XOR 校验错、状态机异常等）。
